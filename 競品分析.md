# 競品分析｜家庭旅遊分帳 App

**分析日期**：2026-06-26  
**分析目的**：找出市場缺口，確認我們的差異化方向  
**資料來源**：Rozi 實際使用截圖 + 使用者訪談

---

## 主要競品：Tricount（by bunq）

### 產品定位
歐洲金融科技公司 bunq 旗下的分帳功能。核心是 bunq 銀行業務（eSIM、免費虛擬卡、支付請求），分帳是其延伸服務，不是主產品。

### Rozi 使用情境
濟州島旅遊（2026年2月）實際使用。旅途中持續使用 Tricount 記帳，但結束後無法匯出、無法結算，仍需回到 Excel 重新整理才能完成最終分帳——旅途記帳的問題解決了，但回台後的結算流程痛點沒有解決。

### 功能盤點

| 功能 | 有無 | 備註 |
|------|------|------|
| 多人共同記帳 | ✅ | 邀請連結加入 |
| 多幣別支援 | ✅ | 自動匯率換算 |
| 自動類別判斷 | ✅ | 根據品項名稱自動比對 emoji，可手動覆蓋 |
| 自訂分帳金額 | ✅ | 可指定每人金額 |
| 餘額計算 | ✅ | 顯示每人應付／應收 |
| 照片分享 | ✅ | 行程內可上傳照片 |
| 開支／收入／轉帳分類 | ✅ | 三種記帳類型 |
| 自訂類別 | ✅ | 可新增自定義類別（含 emoji） |
| **匯出功能** | ❌ | **無法匯出任何格式** |
| **結算標記** | ❌ | **無法標記已付清** |
| **幣別切換顯示** | ❌ | **只能看外幣或台幣，不能自由切換** |
| 唯讀分享連結 | ❌ | 無 |
| 離線支援 | 不確定 | 未測試 |

### 商業模式
- 免費功能：基本分帳
- 付費功能（bunq 訂閱）：支付請求、免費虛擬卡
- 問題：付費功能是銀行業務，與分帳用戶需求錯位

### Rozi 的痛點
1. **無法匯出**——回台後仍需手動整理 Excel 結算表
2. **無法標記結清**——不知道誰付了、誰沒付
3. **幣別無法切換**——想同時看外幣和台幣做不到
4. **商業模式混亂**——付費功能是銀行服務，不是分帳工具

## 次要競品：Splitwise

### 資料來源
Tricount 內建「從 Splitwise 匯入」功能，表示兩者用戶重疊。未取得 Rozi 實際使用截圖，以市場資訊為主。

### 已知特點
- 全球最大分帳 app，主打長期帳款追蹤（朋友、室友、伴侶）
- 支援匯出（PDF）
- 有結算記錄功能
- 不以旅遊情境為主，介面相對複雜
- 免費版有廣告和功能限制，付費版 $4.99/月

## 差異化機會總結

| 維度 | Tricount | Splitwise | 我們的機會 |
|------|----------|-----------|------------|
| 目標場景 | 通用分帳 | 長期帳款 | **專為旅遊設計** |
| 匯出 | ❌ | PDF only | **Excel/CSV + 分享連結** |
| 結算標記 | ❌ | ✅ | ✅ |
| 幣別顯示 | 擇一 | 擇一 | **外幣 + 台幣同時顯示** |
| 分享給家人 | 無唯讀連結 | 無唯讀連結 | **唯讀連結，無需登入** |
| 付款方式記錄 | ❌ | ❌ | **現金／刷卡獨立欄位** |
| 行前費用分段 | ❌ | ❌ | **行前／旅途分段顯示** |
| 商業模式 | 銀行服務 | 訂閱制 | **Phase 2 規劃** |

## 設計借鑒

- **自動類別判斷**（來自 Tricount）：品項名稱自動比對 emoji，可手動覆蓋——保留此設計原則
- **預設自動、允許覆蓋**：不只類別，匯率、成員選擇皆適用
