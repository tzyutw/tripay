# 家庭旅遊分帳 App｜MVP 功能清單

**版本**：Phase 1 MVP  
**確認日期**：2026-06-26  
**拍板日期**：2026-06-26（Rozi 正式確認）  
**決策者**：Rozi  
**技術架構**：PWA｜GitHub Pages + Supabase｜Google 帳號登入

---

## 一、帳號系統

| 功能 | 說明 |
|------|------|
| Google 帳號登入 | 一鍵登入，無需另設密碼 |
| 個人資料 | 顯示名稱、大頭貼（來自 Google） |
| 雲端同步 | 資料存於 Supabase，換裝置不遺失 |

---

## 二、行程管理

| 功能 | 說明 |
|------|------|
| 建立行程 | 行程名稱、emoji 圖示、旅遊幣別、出發日、回程日、成員清單 |
| 編輯行程 | 可修改行程名稱、emoji、幣別、出發／回程日、成員清單（可補加成員） |
| 成員設定 | 輸入成員名稱 + emoji（不需要帳號），Phase 1 由記帳者自行管理 |
| 行前費用 | 行程建立後即可記帳，不限出發日期 |
| 行程狀態 | 進行中 → 已結算 → 封存（三個階段） |
| 多行程管理 | 同一帳號可建立多趟旅遊，各自獨立 |

## 三、記帳（新增／編輯消費）

| 欄位 | 說明 |
|------|------|
| 標題 | 消費品項名稱，自動比對 emoji 類別，可手動修改 |
| 外幣金額 | 旅遊當地幣別，可待填 |
| 台幣金額 | 可待填（現金付款時常回台後才換算） |
| 匯率 | 自動帶入，可手動修改；外幣與台幣可各自獨立輸入，匯率為回推結果 |
| 付款方式 | 現金 / 信用卡 / 儲值卡（Wowpass、Suica 等）（必填，三選一） |
| 備註 | 選填，供用戶記錄刷哪張卡或其他補充說明 |
| 日期 | 預設今日，可修改 |
| 支付者 | 從成員中選擇 |
| 費用類型 | **共同**：選定成員均分（可只勾部分人）／**各自**：同一張單、每人金額不同，逐一輸入／**自付**：不進分帳 |
| 分帳成員 | 可勾選部分成員，未勾選者不參與此筆分帳 |
| 各自金額 | 費用類型選「各自」時，逐一輸入每位成員的實際金額 |

**「稍後補填」機制（統一處理金額不確定情境）：**
外幣金額、台幣金額皆可選「稍後補填」，適用於：
- 現金付款，台幣匯率回台後才決定
- 刷卡付款，等待銀行推播台幣金額
- 任何當下不確定金額的情境

待填欄位在消費列表顯示「⏳ 待填」提示，提醒記帳者補完資料。

**其他邊界處理：**
- 各自金額加總不等於總額 → 顯示警示，但允許儲存（模糊分帳）
- 重複輸入同一筆 → 顯示警示，不阻擋
- 匯率：外幣與台幣金額皆填寫後自動回推，不需手動輸入

## 四、消費瀏覽

| 功能 | 說明 |
|------|------|
| 依日期分段列表 | 出發日前歸為「行前」區段；旅途中依天分組（Day 1、Day 2…） |
| 雙幣別顯示 | 每筆同時顯示外幣金額＋台幣金額 |
| 總覽數字 | 我的開支、總支出（台幣） |
| 快速編輯 | 點擊任一筆可編輯或刪除 |

---

## 五、結算

| 功能 | 說明 |
|------|------|
| 自動計算 | 依每人付出與應分攤金額，計算每人應付／應收台幣金額 |
| 結算摘要 | App 內顯示「Ning 應收 $X，Ziyu 應付 $X…」 |
| 標記結清 | 逐一標記每位成員已付款 |
| 結算後可修改 | 發現記錯可重新開啟，修改後重新計算 |

---

## 六、分享

| 功能 | 說明 |
|------|------|
| 文字摘要 | 一鍵複製結算摘要文字，貼到 LINE 群組 |
| 唯讀分享連結 | 產生連結，任何人無需登入即可瀏覽完整消費明細與結算結果 |
| 即時更新 | 連結內容反映最新資料，修改後自動更新 |
