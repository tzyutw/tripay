## G-02｜S-00 幽靈行程預覽卡（Ghost Trip Card）

**位置**：S-00 登入頁，Google 登入按鈕下方

**用途**：新用戶尚未登入時，展示一張模糊化的「示意行程卡」，讓用戶在登入前就能感受產品質感，降低陌生感。

### 視覺規格

| 屬性 | 規格 |
|------|------|
| 尺寸 | 寬 calc(100% - 48px)，高 88px，圓角 16px |
| 背景 | 白色 #FFFFFF，opacity 0.15 |
| backdrop-filter | blur(8px) |
| 邊框 | 1px solid rgba(255,255,255,0.25) |
| 內容模糊 | 卡片內文字、emoji 以 filter: blur(3px) 呈現 |
| 動效 | 卡片整體 opacity 0.6，pulse animation（1.8s 緩慢呼吸，opacity 0.4→0.7→0.4） |

### 元素結構（模糊化示意）

```
[🏔️]  北海道●●●    進行中 ●
       2025/●●/●● – ●●/●●
       🍋🐟🐵🐱        ⏳ ● 筆待填
```

### 文字覆蓋層

卡片上方疊一層文字說明（不模糊）：

| 屬性 | 規格 |
|------|------|
| 文字 | 「登入後開始你的第一趟」 |
| 字型 | Inter 12px，weight 600 |
| 顏色 | rgba(255,255,255,0.85) |
| 位置 | 卡片正上方，margin-bottom 6px |

### 設計原則

- 幽靈卡不能「太真實」——模糊程度需讓用戶知道這是示意，而非真實資料
- 僅在用戶未登入時顯示，登入後消失
- pulse 動效需輕微，不搶 CTA 的視覺焦點

## G-06｜S-06 頁尾下載 CTA 強化區塊

**位置**：S-06 唯讀分享頁最底部，現有「由旅遊分帳 App 產生 · 唯讀頁面」文字替換為強化版區塊

**用途**：未登入的分享頁瀏覽者（家人）看完消費明細後，引導下載 Tripay

### 視覺規格

| 屬性 | 規格 |
|------|------|
| 區塊背景 | var(--hdr) #7C2D12 |
| 圓角 | 16px（區塊獨立，非全寬） |
| 邊距 | margin 16px，padding 20px |
| 寬度 | calc(100% - 32px) |

### 元素結構

```
[App Icon 佔位 48×48]
Tripay
每一趟，都記得
─────────────────
[想自己記帳？免費下載 Tripay →]   ← 主 CTA 按鈕
```

| 元素 | 規格 |
|------|------|
| App Icon | 48×48px，圓角 12px，暫以 ✈️ emoji 佔位 |
| App 名稱 | Inter 18px，weight 800，#FFFFFF |
| Slogan | Inter 13px，weight 400，rgba(255,255,255,0.75) |
| 分隔線 | 1px solid rgba(255,255,255,0.15)，margin 14px 0 |
| CTA 按鈕 | 全寬，height 48px，bg #FFFFFF，color var(--hdr)，圓角 12px，字 15px weight 700 |
| CTA 文案 | 「想自己記帳？免費下載 Tripay」（Copywriter v1.1 定稿） |

### 設計原則

- 深色背景與上方米白頁面形成明確區隔，讓用戶知道「頁面到底了」
- CTA 白底深字，對比強，點擊意圖清楚
- 不加第二個按鈕，一個 CTA 就夠

## G-08｜S-05 旅程 Highlights 數據卡

**位置**：S-05DONE（全員付清畫面）的「最終結算」綠色區塊下方，封存前展示

**用途**：用數字回顧這趟旅遊的亮點（總天數、消費筆數、最大一筆），強化記憶感與成就感，呼應「旅遊日記」核心敘事

### 視覺規格

| 屬性 | 規格 |
|------|------|
| 卡片背景 | var(--chip) #F5EBD9 |
| 圓角 | 16px |
| 邊距 | margin 0 16px 28px，padding 16px |
| 標題 | 「這趟的紀錄」，Inter 10px，weight 800，color var(--mid)，uppercase，letter-spacing 0.7px |

### 數據欄位（三格橫排）

```
┌──────────┬──────────┬──────────┐
│  8 天    │  34 筆   │ NT$32,490│
│  旅遊天數 │  消費記錄 │ 最大一筆  │
└──────────┴──────────┴──────────┘
```

| 元素 | 規格 |
|------|------|
| 數字 | Inter 22px，weight 800，color var(--ink) |
| 單位標示（天/筆） | 跟在數字後，Inter 12px，weight 600，color var(--mid) |
| 標籤 | Inter 10px，weight 600，color var(--mid)，margin-top 2px |
| 格與格之間 | 1px solid var(--border) 垂直分隔線 |

### 數據來源（Tech Lead 確認）

| 欄位 | 計算方式 |
|------|---------|
| 旅遊天數 | 回程日 - 出發日 + 1（含頭含尾） |
| 消費記錄 | COUNT(expenses) WHERE trip_id = X |
| 最大一筆 | MAX(twd_amount) WHERE trip_id = X（不含待填） |

### 設計原則

- 數字大、標籤小，瞬間抓住眼球
- 暖米色背景與完成感（綠色）結算區形成層次，不搶焦
- 不顯示個人數據，只顯示整趟旅遊維度（避免引發比較）
- 最大一筆金額若有多筆並列，只取第一筆

## Aria 審查備注

G-02 的 blur + opacity 組合在 Safari iOS 上需測試（backdrop-filter 支援度）。  
G-06 的深色區塊要確認不會被誤認為 App footer 導航列，高度控制在 160px 以內。  
G-08 的「最大一筆」欄位若超過 NT$100,000，數字會溢出三欄格——Tech Lead 需處理截斷或縮字邏輯。
