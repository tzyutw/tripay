# 家庭旅遊分帳｜Phase 1 MVP 畫面地圖

**產出者**：UX Lead  
**審查**：Senior UI Lead Aria  
**日期**：2026-06-26  
**依據**：旅程地圖（旅程零–三）、MVP 功能清單 v1、視覺方向定案（旅遊日記精修版）

---

## 一、畫面總覽

| 畫面 ID | 畫面名稱 | 用途一句話 | 是否需要登入 |
|--------|---------|-----------|------------|
| S-00 | 啟動／登入頁 | App 首次開啟或未登入時的入口 | 否（本身就是登入畫面） |
| S-01 | 主頁（行程列表） | 所有行程的總覽，App 的起點 | 是 |
| S-02 | 新增／編輯行程（Sheet） | 建立新行程或修改既有行程資訊 | 是 |
| S-03 | 行程主頁（消費列表） | 某一趟旅遊的核心操作頁，含即時統計 | 是 |
| S-04 | 新增／編輯消費（Sheet） | 新增一筆消費或修改既有消費 | 是 |
| S-05 | 結算頁 | 顯示每人應付應收、標記已付清 | 是 |
| S-06 | 唯讀分享頁 | 無需登入即可瀏覽的消費明細與結算摘要 | 否 |
| S-07 | 個人設定 | 顯示 Google 帳號資訊、登出 | 是 |

**共 8 個畫面**（其中 S-02、S-04 為 Bottom Sheet，非全頁面堆疊）

## 二、畫面詳細說明

---

### S-00｜啟動／登入頁

**用途**：App 首次安裝或尚未登入時顯示，提供 Google 登入入口。

| 項目 | 說明 |
|------|------|
| **進入方式** | App 冷啟動且無有效 session |
| **可前往** | S-01（登入成功後自動跳轉） |
| **UI 狀態** | 單一狀態，無變化 |
| **關鍵元素** | App 名稱與 tagline、Google 登入按鈕、版本號 |

---

### S-01｜主頁（行程列表）

**用途**：登入後的起點，列出所有行程，按建立時間倒序排列。

| 項目 | 說明 |
|------|------|
| **進入方式** | S-00 登入成功後；任何畫面點擊 App logo／返回最上層；S-03 點擊返回 |
| **可前往** | S-02（點「＋ 新增行程」）、S-03（點任一行程卡片）、S-07（點右上角頭貼） |
| **UI 狀態** | 見下表 |

**行程卡片 UI 狀態（不產生獨立畫面，僅改變卡片樣式）：**

| 狀態 | 顯示差異 |
|------|---------|
| 進行中（有待填） | 顯示⏳ N 筆待填角標，橘紅色 |
| 進行中（無待填） | 正常暖色卡片，顯示出發日 + 成員 emoji |
| 已結算 | 綠色勾選標記，顯示結算完成日 |
| 封存 | 卡片降低飽和度（灰階暖調），標示「已封存」 |

---

### S-02｜新增／編輯行程（Sheet）

**用途**：建立新行程（旅程零）或修改行程資訊（名稱、幣別、日期、成員）。

| 項目 | 說明 |
|------|------|
| **進入方式** | S-01 點「＋ 新增行程」（新增模式）；S-03 點右上角「⚙ 設定」（編輯模式） |
| **可前往** | 關閉 Sheet → 返回原畫面（S-01 或 S-03）；建立成功 → 自動進入 S-03（新行程） |
| **UI 狀態** | 新增模式 / 編輯模式（欄位預填既有資料） |
| **關鍵欄位** | 行程名稱、emoji 封面、幣別、出發日／回程日、成員清單（名稱＋emoji）、「這是我」標記（`owner_member_id`） |

**注意**：「這是我」標記僅在**新增模式**強調引導；編輯模式可修改但不再主動引導。

---

### S-03｜行程主頁（消費列表）

**用途**：某一趟旅遊的核心操作頁，包含即時統計卡、依天分組的消費列表、結算入口。

| 項目 | 說明 |
|------|------|
| **進入方式** | S-01 點行程卡片；S-02 建立新行程成功後；S-04 關閉 Sheet 後；S-05 關閉後 |
| **可前往** | S-04（點「＋ 新增消費」或點任一消費筆記進入編輯）、S-05（點「結算」按鈕）、S-02（點「⚙ 設定」）、S-06（點右上角「分享」→ 複製連結或開啟）、S-01（返回） |
| **UI 狀態** | 見下表 |

**S-03 行程狀態（不產生獨立畫面，改變頁面內容與可用操作）：**

| 行程狀態 | 即時統計卡 | 消費列表 | 底部操作區 |
|---------|-----------|---------|-----------|
| **進行中（有待填）** | 統計卡顯示「⚠️ 含 N 筆待填，數字僅供參考」（方案 C，已確認） | ⏳ 橘紅左邊框標示待填筆 | 顯示「＋ 新增消費」＋「前往結算」；右上角工具列常駐「分享」＋「⚙ 設定」 |
| **進行中（無待填）** | 正常顯示全行程花費、我的花費 | 正常列表 | 顯示「＋ 新增消費」＋「前往結算」；右上角工具列常駐「分享」＋「⚙ 設定」 |
| **已結算** | 顯示最終結算日期、總花費 | 列表唯讀（點擊仍可編輯，重新計算） | 顯示「查看結算」＋「封存行程」；右上角工具列常駐「分享」＋「⚙ 設定」 |
| **封存** | 顯示封存日期、總花費 | 列表唯讀 | 無底部操作按鈕；右上角工具列常駐「分享」＋「⚙ 設定」 |

**即時統計卡元素**：全行程總花費（台幣）、我的花費、成員 emoji 橫排切換視角（純前端計算）

---

### S-04｜新增／編輯消費（Sheet）

**用途**：記錄一筆消費，支援所有費用類型與「稍後補填」。

| 項目 | 說明 |
|------|------|
| **進入方式** | S-03 點「＋ 新增消費」（新增模式）；S-03 點任一消費筆記（編輯模式） |
| **可前往** | 儲存 / 取消 → 關閉 Sheet，返回 S-03 |
| **UI 狀態** | 新增模式 / 編輯模式；費用類型切換（共同 / 各自 / 自付）改變表單版型 |
| **關鍵欄位** | 標題（含 emoji 自動比對）、外幣金額（可稍後補填）、台幣金額（可稍後補填）、付款方式、日期、支付者、費用類型、分帳成員、各自金額（費用類型＝各自時展開） |

**表單內部狀態（非獨立畫面）：**

| 費用類型 | 表單版型變化 |
|---------|------------|
| 共同 | 顯示分帳成員勾選；即時顯示「每人 ¥X」 |
| 各自 | 展開每位成員的個別金額輸入欄；即時顯示加總比對 |
| 自付 | 隱藏分帳成員區塊；不計入任何分帳計算 |

---

### S-05｜結算頁

**用途**：顯示每人應付／應收台幣金額，提供標記已付清操作。

| 項目 | 說明 |
|------|------|
| **進入方式** | S-03 點「前往結算」或「查看結算」 |
| **可前往** | S-03（返回，可繼續修改）；分享（複製文字摘要，不跳頁） |
| **UI 狀態** | 見下表 |

**S-05 結算狀態（不產生獨立畫面）：**

| 狀態 | 說明 |
|------|------|
| **未執行結算** | 顯示「執行結算」按鈕；若有待填筆，按鈕上方顯示警示 |
| **已結算（部分付清）** | 顯示每人應付應收；已付清者有✅劃線標記；未付清者有「標記付清」按鈕 |
| **已結算（全員付清）** | 顯示✨全員結清畫面；提供「封存行程」按鈕 |

**結算結果顯示邏輯**（含計算依據展開）：  
主要資訊：「A 應付給 B：$X」大字  
次要資訊：可展開查看「每人付出 vs 應分攤」明細（供核對用）

---

### S-06｜唯讀分享頁

**用途**：無需登入即可瀏覽的行程消費明細與結算摘要，透過唯讀連結開啟。

| 項目 | 說明 |
|------|------|
| **進入方式** | 唯讀連結（由 S-03 或 S-05「分享」產生）；任何裝置瀏覽器直接開啟 |
| **可前往** | 無（純瀏覽，無任何操作按鈕） |
| **UI 狀態** | 單一狀態；連結內容即時反映最新資料（無需重新分享） |
| **關鍵元素** | 行程名稱＋日期＋成員 emoji、結算摘要（每人應付應收）、消費列表（依天分組，雙幣別顯示）、App 下載引導（小字，非主要） |

**注意**：S-06 移除所有 CTA 操作按鈕（無新增、無編輯、無結算），只保留瀏覽內容。

---

### S-07｜個人設定

**用途**：顯示 Google 帳號資訊，提供登出入口。Phase 1 維持極簡。

| 項目 | 說明 |
|------|------|
| **進入方式** | S-01 點右上角頭像 |
| **可前往** | S-01（返回）；S-00（登出後） |
| **UI 狀態** | 單一狀態 |
| **關鍵元素** | Google 大頭貼、顯示名稱、Email、登出按鈕 |

## 四、UI 狀態 vs 獨立畫面判斷總結

| 問題 | 結論 |
|------|------|
| 行程「進行中 / 已結算 / 封存」是否各一個畫面？ | **否**，同為 S-03，狀態改變按鈕與可用操作 |
| 「待填」狀態是否獨立畫面？ | **否**，S-03 列表項目以橘紅左邊框區分；S-05 結算頁以警示 banner 提示 |
| 「新增消費」vs「編輯消費」是否各一畫面？ | **否**，同為 S-04 Sheet，模式不同但元件相同 |
| 「新增行程」vs「編輯行程」是否各一畫面？ | **否**，同為 S-02 Sheet |
| 結算「未結算 / 已結算 / 全員付清」各一畫面？ | **否**，同為 S-05，狀態改變顯示內容 |
| 唯讀分享頁是否獨立畫面？ | **是**，S-06，無需登入，獨立 URL |

## 五、Aria 審查意見（Senior UI Lead）

### ✅ 確認通過的設計決策

| 設計決策 | Aria 確認 |
|---------|---------|
| S-02、S-04 採 Bottom Sheet（非全頁堆疊）| ✅ 符合旅途中快速操作需求；sheet 在手機上操作摩擦最低 |
| S-03 即時統計卡固定於頁面頂部 | ✅ 常駐資訊不需要捲動即可看到，符合「速度優先」原則 |
| S-05 結算頁可從 S-03 進入後仍可返回修改 | ✅ 解決「怕結算後回不去」的焦慮，符合旅程二設計原則 |
| S-06 唯讀頁移除所有操作按鈕 | ✅ 確認；App 下載引導保留小字即可，不搶主要閱讀動線 |
| 8 個畫面數量 | ✅ Phase 1 規模合理，未過度碎片化也未過度合併 |

### 🔶 Aria 補充設計建議（wireframe 參考）

1. **S-03 行程主頁的「＋ 新增消費」按鈕**：建議使用懸浮按鈕（FAB）固定在右下角，即使列表滾動也隨時可觸及；符合 44pt Touch Target 標準
2. **S-02 成員設定區塊**：建議成員卡片可拖曳排序（Phase 1.5），Phase 1 先支援新增／刪除即可
3. **S-04「各自」模式展開**：各成員輸入欄展開時，建議頂部固定顯示「已填加總 vs 總額」比對列，讓用戶不需捲動就能確認
4. **S-05 結算頁計算依據**：「每人付出 vs 應分攤」明細建議以可展開 accordion 呈現，預設折疊；避免第一眼資訊過載
5. **S-06 唯讀頁 header**：行程名稱 + emoji + 日期範圍即可，去掉 App logo（連結分享出去後，品牌曝光留 footer 小字處理）

### ✅ Aria 設計疑問（已確認）

> **S-03「分享」入口位置**：進行中也可分享，移到右上角工具列與 ⚙ 設定並列，全行程狀態常駐。（2026-06-26 Rozi 確認）  
> S-03 行程狀態表格已更新，底部操作區不再包含分享按鈕。
